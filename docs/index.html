λ言語 — kotoba-clj & safe-clj

kotoba は DB であると同時に 言語。kotoba-clj は Clojure/EDN サブセットを 本物の WebAssembly にコンパイルし（インタプリタではなくコンパイラ）、kotoba:kais ホスト世界（graph 読書 / stream / LLM 推論 / CACAO）上で走る Component になる。

safe-clj（compile_safe_clj）は untrusted / AI 生成コードを 能力で封じるプロファイル。思想は「強い言語」ではなく 「渡されていないものには手が届かない実行環境」。3 つの deny-by-default ゲートをコンパイル時に強制する:

① Capability — T3

policy.rs: module の wasm import section ⊆ policy の許諾。未許諾のホスト能力は 生成バイト列に物理的に存在しない → runtime は bind すらできない。能力は名前で呼ぶのではなく 値として渡す（CACAO の attenuation をコンパイル時へ持ち上げたもの）。

② Subset

subset.rs: eval / 実行時 require / 動的 var 変更（set!・binding）/ reflection / 無制限 defmacro を拒否。legacy path が黙って捨てる構文を、安全モードは明示的に弾く。

③ Effect — T2

effects.rs: 関数は宣言 {:effects …} 外の effect を起こせない。 手続き間で検査（write は helper の陰に隠れられない、相互再帰も収束）= Effect Soundness。

監査

embedded_capability_ifaces(wasm)（バイト列の能力面）と infer_effects(src) （ソースの推移的 effect）。CLI kotoba-clj safe-build cell.clj --policy p.edn が両方を報告する。

設計 ADR: ADR-safe-capability-language.md ／ コンパイラ ADR: ADR-clojure-wasm.md

① 書込

QuadStore::assert_datom が 5-tuple を pending に積み、commit() が 4 索引の Prolly Tree を並列ビルド（blake3(key)&0xFF==0 境界 + path-copy）。CARv1 1束で単一 PUT、Commit{index_roots,prev,seq} を CommitDag に append。

② クエリ

4 索引が tier-1。BGP は直接索引スキャン（EAVT 点引き ~180ns）、繰返しは MaterializedView が commit Δ を増分維持。SPARQL は同じ projection 上の補助 RDF 面。as-of は TEA で時間遡行。

③ ブロックストア

kotoba 自身が IPFS block store 兼 pinner（pin/add RPC 不要 → ~35× ingest）。封印 commit を Kubo IPFS と B2 (CAR) へ非同期 export。

④ アンカー

Datom log が canonical state。IPNS 署名ヘッドが per-graph root を固定、Base L2 が CommitDag root を tamper-evidence、AT-Proto MST は ingress/interop。

③Security — 説明可能な秘匿

主権鍵 × ケイパビリティ × E2E 暗号 × X-Road 流アカウンタビリティ。

CACAO 認可

  • 5段検証: 時間 / 能力 / graph scope / 署名 / audience + nonce
  • 能力: datom:read / datom:transact / kotobase:pin
  • depth-2 委譲（減衰のみ）・multi-graph grant
  • 可視性: Private / Authenticated / Public

Signal E2E + custody

  • X3DH 鍵共有 → Double Ratchet（毎メッセージ鍵前進）
  • AES-256-GCM、値は signal:v1: Enveloped datom
  • 暗号文のみ複製 + アクセスレシート + Base L2 anchor
  • block key を t-of-N Shamir 分散（無記帳解放は slashable）

詳細: SECURITY-ARCHITECTURE.md（R0–R3 脅威モデルと層）／ ADR-sealed-cold-tier
