🌷 kotobaってなに？ 🌷

🛡️

とっても安全！

「能力安全」プロファイルで、AI生成コードでも安心して使えます。モジュールが触れるのは、明示的に渡されたものだけ。それ以外は絶対に触れられません！

🚀

WASMにコンパイル

kotobaソースは、本物のWebAssemblyにコンパイルされます。インタープリタではなく、コンパイラ！速くて、小さくて、どこでも動きます。

📊

データモデルも素敵

Datomicスタイルのデータモデルを搭載。[e a v]形式のストアで、データの整理もとっても簡単・キレイ。

✨ kotobaのかわいい特徴 ✨

🔒

デフォルトで拒否

「許可されたものだけ」が実行される、deny-by-defaultの設計。攻撃者が明示的に渡されたもの以外に何もできない、それがkotobaの安全性です。

🎯

能力はバリューで

能力は文字列ではなく、明示的な値（GraphReadCap, GraphWriteCapなど）として渡されます。名前を summon して使うことはありません。

🧩

.kotoba 拡張子

正規のソース拡張子は .kotoba。Clojureファミリーの親しみやすいシンタックスで、書きやすい・読みやすい言語です。

🔐

CACAO認証

CACAO（能力認証）ネイティブの認証・認可システム。深度2の委譲チェーンや、マルチグラフの許可もサポートしています。

🎀 はじめよう！ 🎀

kotoba をインストールして、はじめよう ✨
# Homebrew でインストール（macOS / Linux） brew tap kotoba-lang/kotoba brew install kotoba # コントラクトをチェック kotoba check --kind cli-contract --json # Kotobaソースをコンパイル・実行 kotoba run path/to/entry.kotoba # WebAssemblyにコンパイル（能力安全プロファイル） kotoba wasm emit cell.kotoba --policy policy.edn --package-lock lock.edn -o cell.wasm
