まとめ — end to end

クエリは「索引化された Prolly Tree」を辿るだけ

①書込 QuadStore::assert_datom で 5-tuple (E,A,V,T,Added) を pending に。
②commit で EAVT/AEVT/AVET/VAET(+TEA) の Prolly Tree を並列ビルド（blake3(key)&0xFF==0 境界・path-copy）。
③各ノードを DAG-CBOR → sha2-256 → CIDv1(0x71)、子リンクは IPLD tag-42。Commit{index_roots,prev,seq} を CommitDag(=WAL) に append、CARv1 で単一 PUT。
④封印 commit を Kubo IPFS / B2 へ非同期 export・pin、IPNS で head 公開。
⑤クエリは索引選択 → hot Arrangement か ProllyTree::scan_prefix で cold scan（各レベル=block 1 get）→ Datom 復元。
⑥emit_cid で結果も result_cid として content-address（再現・改ざん検知・キャッシュ）。
